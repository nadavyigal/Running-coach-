# Decisions

Use this folder for product and architecture decisions that should survive beyond a single session.

## Decision Record Format
- Date:
- Status: proposed / accepted / superseded
- Context:
- Decision:
- Alternatives Considered:
- Consequences:
- Follow-up:

## Rules
- Record decisions that affect product direction, architecture, data models, integrations, or deployment.
- Keep entries short and concrete.
- Link related specs, QA reports, or PRs when available.
