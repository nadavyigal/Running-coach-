# Specs

Approved feature specs live here.

Use `.agent-os/templates/feature-spec-template.md` for feature specs and `.agent-os/templates/dev-story-template.md` for implementation stories.

## Spec Rules
- One feature per spec.
- Include product intent, user flows, acceptance criteria, technical approach, validation plan, and rollout notes.
- Break implementation into small stories.
- Do not start implementation until the relevant spec or story is approved unless the user explicitly asks for a quick fix.

## Naming
Use date-prefixed kebab-case:

`YYYY-MM-DD-feature-name.md`
