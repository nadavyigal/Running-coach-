# Screen Recording Checklist

**Purpose:** Track your recording progress and ensure you capture everything needed for all video formats.

---

## Pre-Recording Setup

### Environment Preparation
- [ ] Notifications disabled (OS level + browser)
- [ ] Do Not Disturb mode enabled
- [ ] Close unnecessary applications
- [ ] Clear browser cache (if recording web app)
- [ ] Test recording software (do a 10s test recording)

### App/UI Preparation
- [ ] App is running in clean demo state
- [ ] Demo user data loaded (realistic but not personal)
- [ ] No developer tools/overlays visible
- [ ] No console errors in browser (check DevTools)
- [ ] Today's date makes sense in context
- [ ] All demo workouts are realistic (proper paces, distances)

### Recording Software Setup
- [ ] **16:9 Desktop:** Recording at 1920×1080 or higher
- [ ] **9:16 Mobile:** Recording at 1080×1920 or using mobile device
- [ ] Frame rate set to 30fps or 60fps
- [ ] Quality set to highest available
- [ ] Audio recording OFF (unless doing voiceover simultaneously)
- [ ] Cursor highlighting enabled (optional but helpful)

### Visual Settings
- [ ] Screen brightness at 100%
- [ ] App is full screen or properly framed
- [ ] Text is crisp and readable at target resolution
- [ ] Color contrast is good (check on dark mode if applicable)

## Export Settings for Editing

Once all recordings are complete, export with these settings:

### For Video Editing Software Import
- **Format:** MP4 (H.264 codec)
- **Frame Rate:** 30fps (or 60fps if you recorded at 60fps—keep consistent)
- **Bitrate:** High (10-20 Mbps for 1080p)
- **Audio:** None (or separate audio track if recorded)

### File Naming Convention
```
[Screen Number]-[Screen Name]-[Aspect Ratio].mp4

Examples:
01-onboarding-16x9.mp4
01-onboarding-9x16.mp4
05-adaptation-result-16x9-take2.mp4
```

## Quick Reference: What Makes a Good Recording

✅ **DO:**
- Pause 1s before any interaction
- Move cursor/finger slowly and intentionally
- Hold final state for 1-2s
- Keep UI clean and free of clutter
- Use realistic demo data

❌ **DON'T:**
- Rush through interactions
- Show personal information
- Include notifications or pop-ups
- Have dev tools open
- Use extreme or unrealistic data
