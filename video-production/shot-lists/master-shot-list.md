# Master Shot List - Run-Smart Demo Videos

**Purpose:** This is the complete list of UI screens you need to capture for all video formats.

**Key Principle:** Record each screen as a separate 6-12 second clip. You'll assemble them in editing.

---

## The 9 Essential Screens

### 1. [SCREEN: Onboarding] - Goal Selection + Experience Level

**What to Show:**
- Goal selection interface (5K, 10K, Half Marathon, Marathon, General Fitness, etc.)
- Experience level selector (Beginner, Intermediate, Advanced)
- The user selecting a goal and level

**Recording Instructions:**
- Start with both options visible
- Pause 1s
- Tap/click goal (e.g., "5K")
- Pause 0.5s (let selection highlight)
- Tap/click experience level (e.g., "Beginner")
- Pause 1s
- Tap "Continue" or "Create Plan" button
- Pause 1-2s to show transition/loading if applicable

**Duration:** 8-10 seconds
**Aspect Ratios Needed:** 16:9 (desktop) AND 9:16 (mobile)

**Critical Details:**
- Make sure goal and level are clearly visible
- Selection should be obvious (highlight/color change)
- Button taps should be slow and intentional

---

### 2. [SCREEN: Today] - Today's Workout Card

**What to Show:**
- The "Today" view with current day's workout prominently displayed
- Workout details: type (e.g., "Easy Run"), duration/distance, description
- Start/Preview button or similar CTA
- Optional: streak counter or "next workout" preview

**Recording Instructions:**
- Start with Today screen fully loaded
- Pause 2s (let viewer read the workout)
- Optionally: slow scroll down to see full workout details
- Pause 1s
- Optionally: hover over or tap the "Start" button (don't actually start)
- Pause 1-2s

**Duration:** 6-8 seconds
**Aspect Ratios Needed:** Both 16:9 and 9:16

**Critical Details:**
- Workout type and details must be readable
- Use a realistic workout (e.g., "30-minute easy run" or "4 miles easy pace")
- Date should be "today" or current

---

### 3. [SCREEN: Plan/Calendar] - Week View

**What to Show:**
- Weekly calendar view showing 5-7 days
- Different workout types visible (e.g., Easy Run, Intervals, Long Run, Rest)
- Today highlighted or indicated
- Clear visual hierarchy

**Recording Instructions:**
- Start with full week visible
- Pause 2s
- Slow scroll down to reveal next week OR scroll horizontally to next week
- Pause 1-2s at end
- Keep scroll very gentle (1 scroll max)

**Duration:** 8-10 seconds
**Aspect Ratios Needed:** Both 16:9 and 9:16

**Critical Details:**
- All workout labels should be readable
- Use varied, realistic workouts (not all "Easy Run")
- Today should be visually distinct

---

### 4. [SCREEN: Missed Workout] - Showing a Missed Day

**What to Show:**
- A workout that was scheduled but not completed
- Visual indicator: "Missed" badge, grayed out, or similar
- Contextual info (e.g., "Yesterday" or specific date)

**Recording Instructions:**
- Start with calendar or Today view showing the missed workout
- Pause 2s (let the "missed" status be clear)
- Optionally: tap on the missed workout to show detail modal
- Pause 1-2s
- If modal opened, close it or leave it open for next shot

**Duration:** 6-8 seconds
**Aspect Ratios Needed:** Both 16:9 and 9:16

**Critical Details:**
- "Missed" status must be visually obvious (color, badge, label)
- Should feel like a realistic scenario (e.g., Tuesday workout missed, viewing on Wednesday)
- Don't show too many missed workouts (implies failure)—just 1-2

**Alternative Version:** If your app has a "Move Workout" feature instead of just "missed," record that action:
- Show workout → tap "Move" → select new day → confirm

---

### 5. [SCREEN: Adaptation Result] - The Updated Week

**What to Show:**
- The same week view, but NOW with adjustments made
- Workouts re-ordered or modified
- Visual indicator of change (e.g., subtle highlight, "Updated" badge, animation)
- Clear before/after comparison (if possible in a single shot)

**Recording Instructions:**

**Option A (Preferred): Before/After in One Shot**
- Start with "before" state (with missed workout visible)
- Pause 1s
- Trigger adaptation (tap "Update Plan" button, or refresh, or automatic overnight simulation)
- Show brief loading/transition (0.5-1s)
- Reveal "after" state with updated workouts
- Pause 2s on the result

**Option B: Separate Before/After Shots**
- Record "before" (missed workout visible): 4s
- Record "after" (updated plan): 4s
- Editor will cut these together with a transition

**Duration:** 8-12 seconds (includes transition)
**Aspect Ratios Needed:** Both 16:9 and 9:16

**Critical Details:**
- The change must be CLEAR—use visual callouts if possible
- Updated workouts should make logical sense (e.g., long run pushed to weekend)
- This is the MAGIC MOMENT—make it satisfying and obvious

---

### 6. [SCREEN: AI Chat] - Coach Q&A

**What to Show:**
- Chat interface with AI coach
- User asks a realistic question (e.g., "Should I run today?" or "How hard should I push?")
- Coach provides a short, practical answer (2-3 sentences max)

**Recording Instructions:**

**Option A: Type Question Live**
- Start with empty chat or previous conversation
- Pause 0.5s
- Slowly type question: "Should I run today?"
- Pause 0.5s
- Press send
- Show AI "typing..." indicator (1s)
- Answer appears
- Pause 2s to let viewer read answer

**Option B: Pre-Populated Conversation**
- Start with question already visible
- Pause 1s
- Scroll up slightly to show previous context (optional)
- Pause 2s on the answer

**Duration:** 8-10 seconds (Option A), 5-6 seconds (Option B)
**Aspect Ratios Needed:** Both 16:9 and 9:16

**Critical Details:**
- Question should be simple and relatable
- Answer should be SHORT and practical (not a wall of text)
- Example Q: "Should I run today?" / A: "Yes! You're recovered and today's easy run will build your aerobic base. Keep the pace comfortable."

---

### 7. [SCREEN: Recovery/Insights] - Recovery Score or Readiness

**What to Show:**
- Recovery score interface (0-100 scale or similar)
- Key metrics: sleep, HRV, subjective wellness, or similar
- Simple "push vs rest" guidance or color-coded status

**Recording Instructions:**
- Start with recovery screen loaded
- Pause 2s (let viewer see the score)
- Optionally: tap to reveal more detail (sleep breakdown, HRV trend)
- Pause 1-2s on detail view
- Return to main recovery view OR leave on detail

**Duration:** 6-8 seconds
**Aspect Ratios Needed:** Both 16:9 and 9:16

**Critical Details:**
- Score should be realistic (60-85 range is believable)
- Metrics should be clean and readable
- Use green/yellow/red status colors if applicable
- Example: "Recovery Score: 78 - Good to train"

---

### 8. [SCREEN: Progress/Results] - Streak, Completion, Next Workout Preview

**What to Show:**
- Progress visualization (e.g., "5-day streak," "Week 3 of 12," "8 runs completed")
- Optional: chart/graph of weekly mileage or consistency
- "Next workout" preview card

**Recording Instructions:**
- Start with progress view loaded
- Pause 2s
- Optionally: slow scroll to show more stats
- Pause 1-2s
- Tap on "Next Workout" preview if available
- Pause 1s

**Duration:** 6-8 seconds
**Aspect Ratios Needed:** Both 16:9 and 9:16

**Critical Details:**
- Progress should feel achievable (not overwhelming)
- Example: "5 runs completed, 3 this week" not "47 runs, 203 miles"
- Next workout should tie back to the plan

---

### 9. [SCREEN: CTA End Card] - Booking Call to Action

**What to Show:**
- Clean end card with:
  - Run-Smart logo
  - Clear headline: "Book a 15-minute walkthrough"
  - Booking link or QR code
  - Optional: "Beta access available" badge

**Recording Instructions:**
- This is a STATIC SCREEN (no interaction needed)
- Hold for 3-5 seconds
- Should be designed in Figma/Canva or similar (not in-app)

**Duration:** 3-5 seconds (static hold)
**Aspect Ratios Needed:** Both 16:9 and 9:16

**Critical Details:**
- Extremely clean and minimal design
- CTA text must be large and readable
- Use your brand colors
- Include actual booking URL in small text below CTA
- Example text:
  - Headline: "Ready to see Run-Smart in action?"
  - Subhead: "Book a free 15-minute walkthrough"
  - URL: "runsmart.app/demo"

## Recording Order Recommendation

Record in this order for logical flow:

**Session 1: Core Journey (16:9 Desktop)**
1. Onboarding (Screen 1)
2. Today (Screen 2)
3. Plan/Calendar (Screen 3)
4. Missed Workout (Screen 4)
5. Adaptation Result (Screen 5)

**Session 2: Supporting Screens (16:9 Desktop)**
6. AI Chat (Screen 6)
7. Recovery/Insights (Screen 7)
8. Progress/Results (Screen 8)

**Session 3: Mobile Versions (9:16)**
- Repeat Screens 1-8 in mobile view

**Session 4: End Card**
9. Create CTA End Card (design in Figma, export as image)

## Quality Checklist Before Recording

- [ ] App is in a clean state (no dev overlays, console errors, etc.)
- [ ] Demo data is realistic (names, dates, distances make sense)
- [ ] Notifications are turned off
- [ ] Browser/app is full screen
- [ ] Recording software is set to highest quality
- [ ] You have both 16:9 and 9:16 recording setups ready
- [ ] Audio is muted (or you're in a quiet space if testing with sound)
- [ ] Screen brightness is at 100%
- [ ] No personal data visible (use "Demo User" or similar)
