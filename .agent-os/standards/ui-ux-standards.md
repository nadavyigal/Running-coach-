# UI/UX Standards

RunSmart should feel calm, capable, and specific to running. Avoid generic cheap fitness design.

## Product Feel
- Mobile-first.
- Clean and not overloaded.
- Useful for everyday runners, not only advanced athletes.
- AI-first but practical and grounded.
- Trustworthy about training guidance, recovery, and readiness.

## Today Page
- Treat Today as the daily command center.
- Put the next best action near the top.
- Show next workout, readiness, recovery, progress, and coach recommendation clearly.
- Avoid long walls of metrics.
- Use progressive disclosure for advanced details.

## Training Plans
- Make plan structure easy to understand.
- Explain why a workout matters when useful.
- Make adjustments and missed-workout recovery clear.
- Preserve confidence: avoid surprise changes without explanation.

## Run Reports
- Make reports visual and useful.
- Prioritize key insight, effort, pace/HR patterns, route, and next step.
- Avoid overwhelming tables unless the user asks for detail.

## Responsive Design
- Mobile viewport must be checked for UI changes.
- Text must not overlap or overflow containers.
- Controls should be touch-friendly.
- Desktop should remain usable, but mobile is the primary design target.

## Visual Guidance
- Reuse the existing design system and components.
- Avoid clutter, noisy gradients, and decorative elements that do not help comprehension.
- Do not remove existing functionality during polish unless the spec explicitly says so.
