# Feature Planning Workflow

Use this when turning an idea into an approved feature spec and small development stories.

## Read Only
- `AGENTS.md`
- `CLAUDE.md` or `CODEX.md`
- `tasks/lessons.md`
- `docs/product/current-product-state.md`
- `docs/product/product-vision.md` if product direction is unclear
- This workflow
- Relevant templates
- Relevant existing docs/code only as needed

## Steps
1. Restate the idea and outcome.
2. Identify user, problem, success metric, and non-goals.
3. Check product fit against RunSmart standards.
4. Draft or update a Product Brief.
5. Draft a Feature Spec with acceptance criteria.
6. Split into independently testable development stories.
7. Define validation and QA plan before implementation.
8. Update `tasks/todo.md` with the selected story order.

## Output
- Product brief or brief section.
- Feature spec in `docs/specs/` when approved.
- Story list with validation per story.
- Open questions and risks.

## Stop Conditions
- Missing product decision that changes scope.
- Feature would alter schema/integrations without enough technical context.
- User asked for planning only.
