# Self-Improvement Workflow

Use this whenever a session exposes a reusable mistake or better future rule.

## Triggers
- User correction.
- Failed implementation.
- Bad assumption.
- Broken test.
- Repeated mistake.
- Confusing output.
- Overengineering.
- Missed requirement.

## Steps
1. Name the event in one sentence.
2. Describe why it happened.
3. Convert it into a future rule.
4. Add the lesson to `tasks/lessons.md`.
5. Apply the rule immediately if the current task continues.

## Lesson Quality Bar
- Concrete and short.
- Actionable in future tasks.
- Tied to RunSmart repo behavior or user preference.
- Not a vague reminder like "be careful."
